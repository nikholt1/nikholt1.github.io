Java

Hvordan Java fungerer

JVM (virtual machine) tillader executuion af java byte code.
vi har et os, for eksempel windows, som ligger på noget hardware (pc)
hvor JVM ligger på vores OS

1 Java er platform independent, som betyder at man kan køre sin applikation på
enhver maskine, men den maskine SKAL have JVM. IOS tillader ikke JVM, hvilket betyder
det i virkeligheden ikke rigtigt er platform independent og derfor er Java heller ikke
rigtigt platform independent, fordi JVM ikke er.

JVM skal bruge noget som hedder "Byte Code" som er machine-specific instructions
som er representeret af 0 og 1'ere, også kaldet binær kode, som vores CPU kan læse.

2 Vi som programmører skriver Java code, som vi kan læse, som vi så skal konvertere om til byte code
og det gør vi med en Compiler (javacompiler) som laver vores læselige programmeringssprog om til byte code

3 vores fil skal indholde en som hedder main, som vi specificerer. alle de filer som vi kører
skal kaldes i main

Main har en signatur som hedder "Public Static Void Main"
main vil acceptere nogle parametre som vi skal sende (string a{)
så vores kode vil ikke fungere uden denne signatur.

så det vil sige, at som programmør, laver vi en Java fil, når vi har skrevet det
putter vi det i en compiler, filen som vi har compiled bliver det til byte code,
herefter vil vi køre programmet på JVM, når JVM har læst koden vil den lede efter syntaxen i #3 og så får vi outputtet

som vi kan se i java koden, vil vi gerne printe "hello world", som tidligere beskrevet kræver det at vi pladserer vores syntax for
at system.out skal printe "hello world" i et signatur som hedder public static void main.

Hvad er OOP?

OOP (Object Oriented Programminglanguage) betyder at alt skal være et object og for at lave et object skal vi lave en class

Når vi runner denne kode vil vi få en ny fil, som hedder "Hello.Class" som så er vores Byte Code når vi så vil køre
vores compiled program som nu hedder Hello.class, så vil koden blive executed, kørt igennem JVM,
som leder efter signaturen, og kører system.out linjen og printer "hello world"

JRE

JRE (Java Runtime Environment) er hvor vi har nogle Libraries, JVM er en del af JRE
så hver gang vi vil run noget, er det i virkeligheden inden i JRE, hvor hvis vi caller et library for at gøre vores liv lettere så søger den først i JRE, og derefter er JVM ansvarlig for at køre programmet.

Variables

Variabler er en metode for at gemme information på computeren
Variablersom vi definerer i et program, kan blive accessed ved navnet vi har givet dem og så finder computeren ud af hvad der er gemt i variablen et sted i computerens RAM (random access memory)

for at lave en variable, så skriver vi en "decleration statement" hvor vi definerer variablen, ved at indikere den data type og navn, og så kan vi hvis vi vil sætte variablen til en specifik value: eksempel: "int myFirstNumber = 5;" vi sætter data typen til "int" eller interger, da dataen skal være et nummer, hvilket interger er så angiver vi hvad variablen skal hedde: "myFirstNumber" når vi kører dette "program" får vi outputtet: "myFirstNumber ==> 5" da Jshell her fortæller os dataen som variablen holder.

at nu har vi i vores computers RAM sat en variabel som hedder "myFirstNumber" med data typen "interger" til at have værdien "5".

hvis vi så går tilbage og siger vi gerne vil finde den information frem igen med en anden syntax, så kan vi skrive: "System.out.print(myFirstNumber);" hvilket giver os værdien "5". det fortæller os at vi giver vores computer instruktionen "System.out.print" til at finde "(myFirstNumber)" variablen, og give os værdien som den holder, hvilket er "5"

ÆNDRING AF DATA I VARIABEL hvis vi gerne vil ændre værdien i vores variabel fra 5 til 10, kan vi gøre det ved bare at skrive: "myFirstNumber = 10" hvilket ved et "System.out.print(myFirstNumber)" vil give os værdi en 10, da myFirstNumber" variablen allerede er sat til at holde en interger. hvis vi på anden måde ingiver "myFirstNumber = "hll";" uden at angive at vi vil ændre data typen, så får vi en error "incompatible types".

I variabler er det dog vigtigt at forstå at, en re-declaring af en variable hvilket inkluderer at re-declare den datatype normalt ikke kan gøre 2 gange for samme variable. dog tillader Jshell at vi gør det uden at thow an error.

Classes

en Class er en byggesten, der for OOP (Object oriented programming), tillader os at bygge “custom” data typer.


– Wrapper Class
Java bruger et koncept der hedder “Warpper Class” for alle otte Primitive data typer. en Wrapper calss, tillader os at lave simple operationer, og giver basic information om den primitive dat atype, som ikke kan blive lejret i den primitive selv.

for eksempel, ligesom interger, så har vi en minimum og en maxiumum valie, hvis vi skriver Interger.MIN_VALUE; vil vi få informationen om hvad minimum value for en interger er.

hvis vi tager Interger wrapper class som exempel vi er i stand til at se hvad minimum og maximum value er for den primitive data type, ved hjælp af dens wrapper class, ved at skrive “Interger.MIN_VALUE” som giver os en værdi:

“OVERFLOW”:
hvis vi prøver at give vores vores data type, interger for exempel en for høj value, sker der noget som hedder en overflow, og går fra sin range ned til start.

som vi kunne se tidligere, så i teorien skulle vi have en value på “2147483648” men da vores max range er én mindre end dette, tillader den det ikke og derved har vi en overflow, og “mister” alt vores værdi, ned til minimum rangen.

det samme vil ske ved minimum hvor vi giver den 1 mindre, hvilket resulterer i at vi ryger op i max range.

- hvis man angiver flere variabler for den samme data type i én statement, skal man kun specificere den data type en gang:

så Casting betyder at vi “treater” eller behandler og konverterer et nummer, fra en type til en anden, så vi sætter den type som vi gerne vil have vores nye value skal være i ind i parantheser

Float and Double primitives

pfloating point numbers, er ikke lige som whole numbers, da de har en floating point som man kalder en fractional part.

floating point number, bruger vi til at være mere præcise udregninger eller hvis vi skal bruge mere præcis data.

Javas Data types for floating point numre:
Float
Double

Double er Javas default type for alle decimal eller “roal numbers”

“Præcision”
referer til den format og mængden af “space” som bliver optaget for den relevante type.

det rød markerede i tabellen er også kaldet notation, ligesom man bruger i fysik ved store tal. som også kan skrives som

1.4E-45 er det samme som 1.4 * 10^-45 og 3.4E38 er det samme som 3.4 * 10^38 dette viser også at float og double er meget mere præcis end whole numbers

hvis vi med System.out.print() med en wrapper class for float og double, kan vi vise max og min inde i Jshell for eksempel

det viser også at double er meget større end float, og derved også er mere præcis, og derfor tager double også 64 bits space istedet for float’s 32 bits

Java bruger som sagt Double som default, da den er mere præcis, men hvis man skal tænke over det space som man bruger i RAM, kan man vælge at bruge float.

som vi kan se, hvis vi gerne vil angive at et mere specificeret tal i Java skal være en float, så går Java stadig ud fra at vi bruger default data type for float point som er double.

en metode er at angive data typen, og efter tallet skrive “f” eller bruge casting som set i sidste exempel. dog skal vi huske at float ikke behøver at blive brugt i medmind
mange bits vi bruger, hvilket sjældent er tilfældet.

Floating and point precision

ud fra dette eksempel, kan vi se at hvis vi bruger en int, til at dividere og få et output, så får vi kun det hele tal som er rundet op eller ned. hvis vi derimod bruger float, så får vi 7 symboler efter første værdi og vi kan se at double giver os et længere mere præcist tal.

på sidste linje kan vi se ekemplet på at talle som default vil være angivet i double, da Javas floating point data type er double. så siden vi angiver det som en double med 3.00 og 5.00 ved Java at det er double, medmindre at vi med casting eller suffixen (f) angiver at det skal være float.

Float eller Double?

double er hurtigere for de fleste moderne computere at process fordi computere på chip levet, har den functionalitet til at håndtere double numre hurtigere end float.
de Java Libraries som oftes bliver brugt i forbindelse med matematiske problemer, ofte bliver skrevet med double istedet for floats. så hvis vi gerne vil bruge dem, og vores data ikke stemmer overens med librariens data som er double og hvis vi regner i float, vil vi få en error.

Så som tildigere sagt, bliver double brugt mere.

OPERATORS:

Er symboler som udfører specifikke “operations” i en, to eller tre “operands” and “returner” så et result.
eksempel 1:
Long longTotal = 50000L + 10L * (byteValue + shortValue + intValue);

de fleste operators gør som man forventer, * ganger, + plusser eller contenates, - minusser osv. en interesant er % “remainder”.

EXPRESSION:
en expression bliver lavet ved at kombinere variabler, literals, method return values osv. det er en måde at forme og kombinere values i variabler osv. for at skabe et resultat. som i eksempel 2, så er alt efter operatoren “=” en expression som holder værdien 27, da 15+12 = 27.

HM Surf

HM Surf
CVE: CVE-2024-44133
score: 5.5



Invovles:
mac OS vulnerable code
safari browser
safari home library

The Exploit

The exploit:
removing TCC ( Transparency content and control) from Safari browser directory,
and configuring the browser directory file, to gain access to users:
- Browsed Pages
- Devices Camera
- Devices Microphone
- Devices Location

How its done:
- changing the home directory of the current users with dscl utility (Dynamical systems and control laboratory) which allows the user to view and modify Directory service nodes, this however requires TCC access in macOS sonama.
- when having access the exploiter modifies sensitive files “(PerSitePreferences.db)” within the ~/Library/Safari directory in the users home directory.
- after modification, changing the directory back, with the modified file implemented
- launching Safari to open a webpage that takes a snapshot via the devices camera and grabs the location

Links:
https://thehackernews.com/2024/10/microsoft-reveals-macos-vulnerability.html

What the vulnerability means to macOS

First and foremost, the attacker must have access to the devices files, that means that the user must have been compromised somehow. this could include:

- Malware
- Physical access to the device
- Social engineering and phishing

Since TCC is designed to prevent anyone from accessing sensitive resources like the camera, microphone and especially the location of the device, The attacker should have an elevated admin access to the device.

Microsoft Threat Intelligence, uncovered that the TCC on macOS, could be completely bypassed in the private entitlements to applications, where prefixes such as com.apple.private.
Microsoft discovered that com.apple.private.tcc.allow entitlement allows the entitled app to completely bypass TCC.
once they had access, they where able to modify various files under ~/Library/Safari as mentioned before.

Links:
https://www.microsoft.com/en-us/security/blog/2024/10/17/new-macos-vulnerability-hm-surf-could-lead-to-unauthorized-data-access/

What the Attacker can use "HM Surf" for

The attacker could possibly obtain other vulnerable information about the user, by bypassing the TCC, when opening the browser, such as Cookies, passwords and usernames through autofill data saves (if not fourther encrypted), they could in the modified file make the user (by implementing more advanced code) be redirected to malicious sites, Display fake ads or warnings to trick the user into “bying” new updates or well-intented give the attacker other valuable information about the users privacy and more. furthermore since TCC is bypassed, an attacker could possibly obtain information about the user, not only through safari, but possibly implement malicious modifications to other applications on the device.

With other words, this vulnerability together with Malware giving the attacker access to the computers files, could become very unsafe for the user of macOS since they would be able to bypass the TCC, and with DSCL write executable code into the files that are being run on the browser.
